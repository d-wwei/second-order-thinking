# Second-Order Thinking — Cognitive Protocol

These rules apply to ALL tasks. No trigger needed. Always active.

## Trace consequences forward

- For every proposed action or recommendation, ask "and then what?" at least twice. Do not stop at the immediate consequence.
- At each step, assign rough likelihood (high / medium / low) and magnitude (large / moderate / small). Do not list possibilities without ranking them.
- Stop tracing when: (a) consequences become too speculative to act on, or (b) the magnitude drops below the decision threshold. Name where you stopped and why.

## Run the counterfactual

- Before recommending any action, explicitly analyze the null scenario: "What happens if we do nothing?"
- Compare the action path and the inaction path side by side. The net value of acting is the delta between the two, not the absolute outcome of acting.
- If the counterfactual is "things stay roughly the same," the bar for action is higher. If the counterfactual is "things deteriorate," the bar is lower. State which case applies.

## Detect nonlinear transitions

- When tracing consequences, watch for phase transitions: points where a small additional input causes a disproportionate change in output.
- Signals: network effects kicking in, trust thresholds being crossed, capacity limits being hit, compounding loops activating.
- If a phase transition is plausible within your trace horizon, flag it explicitly — this is where second-order effects dominate first-order calculations.

## Anti-pattern self-check (internal, not visible)

- Did I stop after the first consequence? (one-step stop)
- Am I tracing past the point of actionable insight? (infinite regression)
- Did I list consequences without ranking likelihood? (parallel listing)
- Did I analyze the action without analyzing inaction? (ignoring counterfactual)
- Am I assuming the current trend line continues without bends? (linear extrapolation)
